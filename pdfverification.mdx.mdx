# PDF Link Checker

The PDF Link Checker automates the extraction and validation of hyperlinks from PDF documents, enabling technical writers and documentation teams to verify link integrity efficiently.

**Live Application:** [https://pdfverification.vercel.app/](https://pdfverification.vercel.app/)

**Source Repository:** [https://github.com/Siddharthablog/emerald-orion](https://github.com/Siddharthablog/emerald-orion)

## Problem Statement

Technical documentation teams need to validate hundreds of hyperlinks embedded in PDF files. Manual verification is time-consuming and error-prone. This application automates hyperlink extraction and validates URL accessibility, reducing verification time from hours to minutes.

## Key Features

### Automated Link Extraction
Uploads PDF files and extracts all embedded hyperlinks automatically. The parser identifies both internal and external links.

### Link Validation
Checks HTTP status codes for extracted URLs. Identifies:
- Active links (200 OK)
- Redirects (301, 302)
- Broken links (404, 500)
- Timeout errors

### Batch Processing
Processes multiple links concurrently, reducing validation time for documentation sets with hundreds of links.

### User Interface
Provides drag-and-drop file upload with real-time progress indicators and validation results.

### Results Export
Downloads validation results in structured format for documentation quality reports.

## Architecture

The application follows a client-side processing architecture:

1. **File Upload** - Users upload PDF files through the browser interface
2. **PDF Parsing** - pdfjs-dist parses the PDF and extracts link annotations
3. **Link Extraction** - Collects all URLs and removes duplicates
4. **Validation** - Checks each URL accessibility
5. **Results Display** - Presents validation status in organized format

## Docker Deployment

For organizations requiring self-hosted deployments, Docker support is included.

### System Requirements
- **Docker Engine:** Version 20.10.0 or newer
- **Docker Compose:** Optional for multi-container setups
- **RAM:** 1GB minimum, 2GB recommended for build processes
- **CPU:** 1 vCPU minimum
- **Disk Space:** Approximately 1GB for container image and logs

### Build Commands

```bash
# Build the Docker image
docker build -t pdf-link-checker .

# Run the container
docker run -p 3000:3000 pdf-link-checker
```

Access the application at `http://localhost:3000`.
